The complete atomic inner-thread maintenance batch. Use `create` for a new coherent thread, `update` when the same thread changes, `resolve` when it no longer wants reconsideration, and `delete` for mistakes or meaningless duplication. `about` identifies self, community, or one user. `recall.scope` is `guild` for context learned or meaningful only in that guild and `anywhere` only for deliberately general, context-independent inner life. `recall.mode` is `always` or `users`; user presence controls automatic recall, not ownership. `salience` is lasting personal importance; `pressure` is how strongly the thread wants reconsideration now. Keep each thread self-contained. When understanding or continuing it depends on specific conversations, promises, questions, images, or events, include up to three relevant `source_message_ids` in useful order; omit them for a private thought with no message anchor, omit them on update to preserve existing IDs, and use `null` or `[]` to clear them. Never globalize potentially private guild information merely because it matters.
