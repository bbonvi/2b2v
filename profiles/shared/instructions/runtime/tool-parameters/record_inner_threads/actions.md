The complete atomic inner-thread maintenance batch. Use `create` for a new coherent thread, `update` when the same thread changes, `resolve` when it no longer wants reconsideration, and `delete` for mistakes or meaningless duplication. `about` identifies self, community, or one user. `recall.scope` is `guild` for context learned or meaningful only in that guild and `anywhere` only for deliberately general, context-independent inner life. `recall.mode` is `always` or `users`; user presence controls automatic recall, not ownership. `salience` is lasting personal importance; `pressure` is how strongly the thread wants reconsideration now. Keep each thread self-contained. When understanding or continuing it depends on a specific conversation, promise, question, image, or event, include the smallest useful set of `source_message_ids`; a private thought with no message anchor does not need one. Never globalize potentially private guild information merely because it matters.
