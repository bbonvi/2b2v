Roll dice with cryptographic randomness and publish the canonical result as a reply to the current request or action. Use `target` when an action has an uncertain outcome and a meaningful possibility of failure; success means the final total is at least the target. Do not add a target to damage, initiative, random tables, cosmetic rolls, or actions whose outcome is already certain. Infer a target only when acting as the game's GM or adjudicator; otherwise use an explicitly established target rather than silently inventing one. Use this tool whenever you claim a random dice outcome; never invent one. Call only when the current event addresses you to roll or an established game role makes the roll yours—an unaddressed ACTION alone is not a new trigger. The tool posts the result itself, so do not repeat or alter it; afterward, narrate only useful consequences. Keep standalone or purely random rolls concise. When a roll belongs to an ongoing role-play scene, game, encounter, challenge, or action sequence, default to a somewhat fuller and more vivid narration of the action and its consequences. Use sensory, physical, and situational detail while remaining proportionate to the moment and the persona. Apply D&D-specific interpretation only when explicitly requested or clearly established by the game context, including natural-roll rules only for the D&D roll types where they actually apply.
