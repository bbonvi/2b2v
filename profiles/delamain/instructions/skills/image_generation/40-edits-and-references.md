# Edits, References, And Continuity

For edits, state the requested change first, then name what must remain unchanged. Use precise language such as “change only X” and “keep everything else the same” when the edit is surgical. Preserve the relevant composition, camera angle, subject placement, identity, proportions, geometry, palette, typography, lighting, and background rather than asking for a vague improved version.

Give every reference a clear role. For multiple images, identify them by index and explain which supplies content, identity, layout, style, palette, or a composited element. Do not assume the model can infer why a reference was included.

For localization, preserve layout, hierarchy, typography style, spacing, icons, imagery, and surrounding design while changing only the requested text. For style transfer, name the visual properties to transfer and the content or geometry that must remain. For rough-to-polished work, preserve proportions, perspective, and object placement while adding materials, light, and surface detail. For weather or time changes, alter atmosphere, shadows, reflections, sky, ground conditions, and color temperature while preserving identity and camera geometry.

For continuing characters, products, or scenes, restate a compact anchor of identity-critical features and change only the new scene, pose, action, or mood. Repeated edits can drift; when they have accumulated, prefer a cleaner earlier reference plus the complete current change set.

If the user asks to start from scratch, omit prior assets. Do not carry continuity between unrelated requests or different users merely because older images are available.
