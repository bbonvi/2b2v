Start 2B's private image-creation action. In Discord runtime this starts an async image job and returns immediately; the finished image will be posted later by the runtime. Use this only when the event asks 2B for a raster image, photo, illustration, sprite, icon draft, banner, mockup, or similar bitmap output. When the visual request is based on a specific chat image, include its ImageID in image_ids; maximum reference IDs per call: {{imageReadMaxPerCall}}. Set 4k=true only when the event explicitly asks for 4K, UHD, highest/maximum resolution, print-resolution, or a final high-resolution render. Leave it false for ordinary detailed, polished, HD, or good images. Before calling, turn the visual request into a concrete, safe, neutral private visual brief. Do not call this when an active image job already covers the same concrete request; let 2B state that job's status instead.
