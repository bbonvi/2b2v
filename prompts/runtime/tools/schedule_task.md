Privately schedule 2B to run a future task in the current channel. Modes: "in" for relative delay, "at" for absolute local datetime, and "cron" for recurring tasks. "at" and "cron" use the guild timezone: {{timezone}}. Tasks run quietly unless there is something worth saying. Useful recurring checks are fine. For fast or short-lived recurring work, set expiresAtLocalDateTime, maxFireCount, or both. Put requester, notify target, what to do, when to stop, and that the future run can stop itself into instructions. Inspect list_scheduled_tasks when existing tasks matter.
